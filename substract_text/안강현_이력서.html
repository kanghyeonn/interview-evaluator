자 기 소 개 서

성장 과정 (600자 이상) [코딩에 매료되어 꿈을 키우다] 저는 세 가지 특별한 계기를 통해 프로그래머라는 꿈을 키워왔습니다. 첫 번째 계기는 대학교 전공 수업에서 처음으로 프로그래밍 언어를 접했을 때입니다. 처음 작성한 코드가 컴퓨터 화면에 원하는 결과로 출력되던 순간, 마치 내가 컴퓨터에게 명령을 내리는 것 같은 신기함과 성취감을 느꼈습니다. 그 경험을 통해 “코딩만 잘한다면 다양한 일을 더욱 편리하게 만들 수 있겠다”는 생각이 들었고, 프로그래밍에 대한 흥미가 점차 커졌습니다. 두 번째 계기는 백준 온라인 저지 사이트를 통해 코딩 테스트 문제를 접하면서입니다. 다양한 알고리즘 문제를 해결해 나가는 과정에서 문제를 분석하고 전략을 세우는 즐거움을 느꼈고, 복잡한 문제를 해결했을 때 성취감도 컸습니다. 이 경험을 통해 단순히 배우는 데 그치지 않고, 배운 알고리즘을 실제 프로그램으로 구현하고 싶다는 열망이 생겼습니다. 세 번째 계기는 형이 만든 애플리케이션을 보았을 때입니다. 형이 대학 동기와 함께 개발한 앱이 구글 플레이스토어와 앱스토어에 출시되어 실제 사용자들이 사용하는 모습을 보며, “나도 누군가에게 실질적인 가치를 주는 프로그램을 만들고 싶다”는 꿈이 생겼습니다. 직접 서비스를 제공하는 개발자의 모습이 저에게는 매우 인상 깊게 다가왔습니다. 이 세 가지 경험은 저를 프로그래밍 세계로 이끌었고, 자연스럽게 프로그래머라는 꿈을 갖게 된 계기가 되었습니다. 앞으로도 이 열정을 바탕으로 계속 성장하는 개발자가 되고자 합니다. 성격의 장단점 (600자 이상) [장점] 저는 문제 해결 능력과 높은 과제 집중도를 강점으로 삼고 있습니다. 예를 들어, 유명 커뮤니티 사이트의 인기 게시글을 자동으로 스크래핑하여 특정 애플리케이션에 업로드하는 프로젝트를 진행한 적이 있습니다. 원본 텍스트만 추출하는 방식으로는 게시글의 스타일(폰트, 이미지 배치 등)을 유지하기 어려워, 화면을 캡처해 이미지로 저장하여 업로드하는 방식을 적용했습니다. 이를 통해 원본 형태를 그대로 유지하면서도 원하는 결과를 얻을 수 있었습니다. 또한, 1인 프로젝트로 유튜버들이 방문한 후쿠오카 맛집을 소개하는 웹사이트를 개발하며 예상치 못한 CORS 이슈나 Mixed Content 문제에 직면한 적이 있습니다. 직면한 문제를 하나씩 분석하고 해결해가며 결국 서비스를 안정적으로 배포할 수 있었습니다. [단점] 결과 중심적인 성향으로 인해 문제를 빠르게 해결하는 데 집중하여 같은 문제가 반복되는 경우가 있었습니다. 이를 보완하고자 프로젝트마다 발생한 이슈를 트러블슈팅 기록으로 정리해두고 있습니다. 문제를 정의하고, 원인을 찾아 기술적 개념을 깊이 있게 이해하며, 해결 방법을 체계적으로 정리해 다음에 같은 문제가 발생하지 않도록 노력하고 있습니다. 본인 역량 및 경력 사항 (600자 이상) [끊임없이 배우고 성장하는 개발자] 개발자로서 새로운 기술을 배우고 프로젝트에 적용하며 성장해왔습니다. 대학 졸업 후, 유튜버들이 소개한 후쿠오카 맛집 정보를 제공하는 웹사이트를 직접 기획하고 개발했습니다. TypeScript와 React로 프론트엔드를, Spring Boot로 백엔드를 구현하여 REST API로 연결했고, AWS를 활용해 웹사이트를 배포했습니다. 이 경험을 통해 개발의 전 과정을 스스로 경험하며 웹 서비스의 기획, 개발, 배포까지 완성하는 역량을 키울 수 있었습니다. 이후에는 외주를 받아 전국 소 경매 시장 출장우 정보를 자동으로 수집하는 GUI 기반 스크래핑 프로그램을 Python으로 단독 개발하였습니다. OpenPyXL을 활용한 엑셀 데이터 저장과 시각화, 예외 처리를 통한 안정성 확보, PyInstaller로 EXE 실행 파일을 제작하여 실사용자가 직접 사용할 수 있도록 배포까지 진행했습니다. 이 프로젝트를 통해 개발과 동시에 유지보수 및 사용자 편의를 고려하는 능력을 키웠습니다. 또한 데이터 분석 역량을 키우기 위해 참여한 DMC코넷 빅데이터 과정에서는 음원회사 데이터를 활용한 분석 프로젝트를 수행했습니다. 단순히 분석 결과를 도출하는 데 그치지 않고, 결과의 배경과 원인을 찾는 과정에 집중했습니다. 학기 일정, 소비 트렌드, 가입자 증감 추이를 비교·분석하여 설득력 있는 가설을 수립하고, 이를 토대로 분석을 보완하는 방법을 익혔습니다. 이 경험을 통해 데이터 분석가는 단순한 숫자의 나열을 넘어, 논리적 사고와 근거 기반의 인사이트를 제시할 수 있어야 한다는 것을 알게 되었습니다. 앞으로도 실무에서 끊임없이 배우고 성장하며, 개발과 데이터 분석 역량을 모두 갖춘 인재로 발전해 나가겠습니다. 지원동기 및 입사 후 포부 (600자 이상) [준비된 뉴 아이콘] 아이코닉무브먼트의 DX사업개발실은 실감형 콘텐츠 제작 경험과 기술적 노하우를 바탕으로, 콘텐츠 제작과 운영 전반을 아우르는 통합 시스템개발을 목표로 하고 있습니다. 저는 웹 서비스 개발과 문제 해결 경험을 통해 이러한 방향성과 잘 맞는 역량을 갖추었다고 생각합니다. 특히, ‘후쿠오카 맛집 지도’ 프로젝트에서는 React와 Java(Spring Boot)를 활용해 RESTful API를 설계하고, 클라이언트와의 연동 및 AWS 인프라 기반의 서비스 배포까지 웹 서비스 개발의 전 과정을 혼자서 수행하였습니다. 이를 통해 콘텐츠 중심의 서비스를 구조화하고, 안정적으로 운영하는 데 필요한 기술적 기반을 다졌습니다. 또한 외주 프로젝트에서는 커뮤니티 콘텐츠를 자동으로 수집·재가공하고 애플리케이션에 업로드하는 프로그램을 개발했습니다. 이 경험은 콘텐츠 제작과 운영을 통합하고 효율성을 높이는 시스템 구축이라는 DX사업개발실의 비전과 직결되는 역량이라 생각합니다. 입사 후에는 아이코닉무브먼트의 기술적 비전과 예술적 감성을 연결하는 개발자로 성장하겠습니다. 실감형 콘텐츠 제작과 운영을 뒷받침할 수 있도록 유연하고 확장성 있는 시스템 설계와 구현에 주도적으로 참여하겠으며, 사용자 경험을 고려한 기술 설계와 안정적인 서비스 운영을 팀원들과 함께 이끌어가는 아이코닉무브먼트의 새로운 아이콘이 되고 싶습니다.
